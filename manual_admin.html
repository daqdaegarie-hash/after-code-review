🏫1. 시스템 개요 및 관리자 권한

1.1 AFTER 시스템 소개

AFTER(Assistant For Teachers' Educational Requirements)는 경상남도 중·고등학교 교감 선생님의 행정 업무를 AI로 지원하는 웹 시스템입니다. Google Gemini API를 활용한 자연어 질의응답, 1,800개 이상의 서식 자료실, 회원 관리, 게시판 등 다양한 기능을 제공합니다.

🤖 AI 질문/답변

자연어 질문 → Gemini 답변 + 관련 서식 다운로드 버튼 자동 제공

📁 서식 자료실

8개 부서별 HWP·PDF·XLSX 등 1,800+ 서식 관리

👥 회원 관리

가입 승인 2단계 프로세스, VIP·포인트 설정

💾 백업/복원

전체 DB JSON 백업, 클라우드 저장, 복원 지원

✅3. 회원 가입 신청 승인

3.1 2단계 가입 프로세스

보안을 위해 회원가입은 2단계로 처리됩니다. 직접 users 테이블에 쓰는 대신, 먼저 signup_requests 임시 테이블에 저장된 후 관리자 승인 시 users 테이블로 이동합니다.

🔄 전체 흐름

① 사용자: 회원가입 폼 작성 → signup_requests 저장 (status: pending)

② 관리자: 패널에서 신청 목록 확인 → 승인 또는 거부

③ 승인 시: users 테이블로 이동 + 1,000포인트 자동 지급 + 승인일 기준 1개월 후 만료일 설정

④ 거부 시: signup_requests에서 삭제

3.2 승인 절차

1

관리자 패널 열기

사이드바 → "관리자 모드" 클릭

2

"👥 회원 관리" 탭 클릭

상단에 "⏳ 가입 신청 승인 대기 (N건)" 박스가 표시됩니다.

3

신청자 정보 확인

  • 아이디, 이름, 이메일, 학교명, 신청 일시
  • 학교 소속 여부를 확인하여 가짜 가입 방지
4

승인 또는 거부

✅ 승인: "승인" 버튼 클릭 → users 테이블로 이동, 1,000P 지급, 1개월 만료일 설정

❌ 거부: "거부" 버튼 클릭 → signup_requests에서 삭제

✅ 승인 완료 후

사용자는 즉시 로그인 가능하며 초기 1,000 포인트가 자동으로 부여됩니다. (서식 다운로드 100회 또는 AI 질문 200회 분량)

🤖6. Gemini AI API 키 관리

6.1 API 키 필요 이유

AFTER의 AI 질문 기능은 Google Gemini API를 사용합니다. API 키 없이는 AI 기능이 전혀 작동하지 않습니다.

🚨 최우선 설정 항목

배포 후 가장 먼저 API 키를 설정하세요. 설정 전에는 모든 사용자의 AI 질문이 차단됩니다.

6.2 무료 API 키 발급

1

Google AI Studio 접속

https://aistudio.google.com/app/apikey

Google 계정으로 로그인 필요

2

"Create API key" 클릭 → 키 복사

형태 예시: AIzaSy...

3

관리자 패널 → "🔑 AI 설정" 탭

API 키 입력란에 붙여넣기 → "저장하고 연결하기" 클릭

4

연결 테스트

"연결 테스트" 버튼 클릭 → "AI 연결 성공" 메시지 확인

6.3 무료 사용 한도

항목무료 한도
분당 요청15회
일일 요청1,500회
모델gemini-1.5-flash (기본)
💡 한도 초과 시

일시적으로 AI 기능이 차단됩니다. 내일 0시(UTC) 이후 자동 초기화됩니다. 사용량이 많다면 유료 플랜 을 검토하세요.

9.4 게시판 검색 기능

게시판 상단 검색창에서 제목, 내용, 관리자 답변 텍스트로 통합 검색이 가능합니다. 검색어는 목록에서 노란색으로 하이라이트됩니다.

  • 비밀글은 검색 시 제목만 매칭 (내용 비공개 유지)
  • 관리자 답변 내용도 검색 결과에 포함되며 미리보기에 파란색 박스로 표시

💾10. 백업 및 복원

10.1 백업해야 하는 시점

🚨 필수 백업 시점
  • 서식 파일 대량 추가 전
  • 시스템 업데이트 배포 전
  • 학기 초·학기 말
  • 회원 데이터 대량 수정 전

10.2 전체 백업 방법

1

관리자 패널 → "💾 백업/복원" 탭

2

"전체 백업 (모든 테이블)" 버튼 클릭

3

진행률 바 완료 확인

백업 대상: users, notices, board_posts, chat_history, base_forms, user_forms, ai_feedback, app_config

4

JSON 파일 자동 다운로드 + 클라우드 자동 저장

파일명 예: AFTER_fullbackup_20260421_153045.json

💡 file_data 제외

실제 첨부 파일 데이터(file_data)는 용량이 크기 때문에 백업 파일에 포함되지 않습니다. 복원 후 관리자 패널 → 서식DB 탭에서 파일을 다시 업로드하세요.

10.3 클라우드 백업 이력 조회

백업/복원 탭 하단 "☁️ 클라우드 백업 이력" 섹션에서 최근 30건의 백업 이력을 조회할 수 있습니다. 각 항목에서 다운로드가 가능합니다.

10.4 복원 방법

1

배포 URL + /restore.html 접속

2

관리자 비밀번호 입력

3

백업 JSON 파일 업로드

4

❓11. 문제 해결 및 FAQ

11.1 자주 발생하는 오류

❌ 회원가입 시 401 오류

원인

GenSpark Table API는 비인증 사용자의 users 테이블 직접 쓰기를 차단합니다.

해결: 이미 signup_requests 임시 테이블 방식으로 변경 완료됨. 배포 URL에서만 동작하며, 미리보기 URL에서는 일부 제한이 있을 수 있습니다.

❌ AI 질문이 작동하지 않음

해결: 관리자 패널 → AI 설정 → API 키 입력 여부 확인. 키가 있으면 "연결 테스트"로 유효성 확인. 한도 초과 시 내일 재시도.

❌ 복원 후 서식이 표시되지 않음

해결: 백업에는 file_data가 없습니다. 관리자 패널 → 서식DB 탭에서 파일을 다시 업로드하세요. 메타데이터는 복원됩니다.

❌ 게시판 답변이 표시되지 않음

해결: 관리자 로그인 상태 확인. 게시글 상세 화면 하단에 "관리자 답변 작성" 영역이 보여야 합니다. 로그아웃 상태에서는 이미 등록된 답변만 보입니다.

11.3 긴급 복구 순서

🚨 데이터 손실 발생 시
